# ⚠️ Проблема с портом 5000 на macOS

## Проблема

На macOS порт **5000 по умолчанию занят сервисом AirPlay (AirTunes)**. Это системный сервис Apple, который блокирует доступ к этому порту.

Когда вы пытаетесь запустить FastAPI на порту 5000, вместо вашего сервера отвечает AirPlay, который возвращает ошибку 403.

## Решение

Порт изменён на **8000** (стандартный порт для FastAPI/uvicorn).

## Новые адреса:

- **Swagger UI**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc
- **OpenAPI JSON**: http://localhost:8000/openapi.json
- **Главная страница**: http://localhost:8000

## Как изменить порт обратно на 5000 (если нужно):

### Вариант 1: Отключить AirPlay на порту 5000

1. Откройте **Системные настройки** → **Общий доступ**
2. Найдите **AirPlay Receiver**
3. Отключите его или измените порт

### Вариант 2: Использовать другой порт в .env

Создайте или отредактируйте файл `.env` в корне проекта:

```env
PORT=5001
HOST=127.0.0.1
```

Затем перезапустите сервер.

### Вариант 3: Указать порт при запуске

```bash
uvicorn app.main:app --host 127.0.0.1 --port 5001 --reload
```

## Проверка занятости порта

Чтобы проверить, какой процесс использует порт:

```bash
lsof -i :5000
```

Или:

```bash
netstat -an | grep 5000
```

## Рекомендация

Используйте порт **8000** - это стандартный порт для FastAPI и не конфликтует с системными сервисами macOS.
