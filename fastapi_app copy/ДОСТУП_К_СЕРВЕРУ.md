# Доступ к серверу FastAPI

## ⚠️ Важно: Используйте localhost вместо 0.0.0.0

Сервер может быть настроен на прослушивание `0.0.0.0`, но **для доступа в браузере всегда используйте `localhost` или `127.0.0.1`**.

## Правильные адреса:

✅ **Используйте:**
- http://localhost:8000
- http://127.0.0.1:8000

❌ **НЕ используйте:**
- http://0.0.0.0:8000 (браузеры могут блокировать этот адрес)

## Swagger UI:

- **Swagger UI**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc
- **OpenAPI JSON**: http://localhost:8000/openapi.json

## Запуск сервера:

### Вариант 1: С настройками по умолчанию (localhost)
```bash
cd fastapi_app
uvicorn app.main:app --reload
```
Сервер будет доступен на http://localhost:8000

### Вариант 2: Явно указать localhost
```bash
cd fastapi_app
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

### Вариант 3: Для доступа извне (например, с другого устройства в сети)
```bash
cd fastapi_app
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
Но даже в этом случае используйте `http://localhost:8000` или `http://ВАШ_IP:8000` для доступа.

## Проверка работы сервера:

1. Запустите сервер
2. Откройте в браузере: http://localhost:8000/docs
3. Вы должны увидеть Swagger UI с документацией API

## Если получаете ошибку 403:

1. **Убедитесь, что используете `localhost`, а не `0.0.0.0`**
2. Проверьте, что сервер запущен (должны видеть сообщение в консоли)
3. Проверьте, что порт 8000 не занят другим приложением
4. Попробуйте другой порт: `uvicorn app.main:app --host 127.0.0.1 --port 9000 --reload`

## Настройка хоста в .env:

Вы можете изменить хост по умолчанию в файле `.env`:

```env
HOST=127.0.0.1
PORT=8000
```

Или для доступа извне:

```env
HOST=0.0.0.0
PORT=8000
```

Но помните: даже если сервер слушает на `0.0.0.0`, для доступа в браузере используйте `localhost` или `127.0.0.1`.
